1.13. Metadata export

This section explains the new metadata exporter available from DHIS2.23, the metatada API is available at /api/23/metadata and /api/24/metadataendpoints, both XML and JSON is supported in this exporter.

The most common parameters are described below in the "Export Parameter" table, but you can also apply this to all available types by using type:fields=<filter> and type:filter=<filter>, you can also enable/disable export of certain types by setting type=true/false, please see examples of these down below.

Table 1.20. Export Parameter

NameOptionsDescription
fieldsSame as metadata field filterDefault field filter to apply for all types, default is :owner.
filterSame as metadata object filterDefault object filter to apply for all types, default is none.
orderSame as metadata orderDefault order to apply to all types, default is name if available, or created if not.

1.13.1. Metadata export examples

Export all metadata:

curl -u user:pass http://server/api/24/metadata

Export all metadata ordered by lastUpdated descending:

curl -u user:pass http://server/api/24/metadata?defaultOrder=lastUpdated:desc

Export id and displayName for all data elements, ordered by displayName:

curl -u user:pass http://server/api/24/metadata?dataElements:fields=id,name&dataElements:order=displayName:desc

Export data elements and indicators where name starts with 'ANC':

curl -u user:pass http://server/api/24/metadata?defaultFilter=name:^like:ANC&dataElements=true&indicators=true

1.13.2. Data set and program export with dependencies

When you want to move a whole set of data set or program metadata from one server to another (possibly empty) server, we have two special endpoints for just that purpose:

/api/24/dataSets/ID/metadata.json
/api/24/programs/ID/metadata.json

These exports can be imported both using the old importer at /api/metadata and the new importer at /api/23/metadata.
