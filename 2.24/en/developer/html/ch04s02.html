4.2. Technical Requirements

The DHIS2 is intended to be installed and run on a variety of platforms. Hence the system is designed for industry standards regarding database management systems and application servers. The system should be extensible and modular in order to allow for third-party and peripheral development efforts. Hence a pluggable architecture is needed. The technical requirements are:

  • Ability to run on any major database management system

  • Ability to run on any J2EE compatible servlet container

  • Extensibility and modularity in order to address local functional requirements

  • Ability to run on-line/on the web

  • Flexible data model to allow for a variety of data capture requirements
